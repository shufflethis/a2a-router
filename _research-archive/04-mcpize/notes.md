## 🔎 Desk-Research-Snapshot (von Claude vor dem Sprint)

**Positionierung:** *"MCP: Like API for AI Agents. MCP Marketplace. 650+ Model Context Protocol servers for Claude, Cursor, and VS Code. Publishers keep 80% of every subscription."* — **Monetisierung als Hauptdifferenzierung.**

**Was sie machen:**
- **650+ MCP-Server** (deutlich kleiner als Glama mit 23K)
- **80% Revenue Share** für Publisher — pitcht offen, dass andere Plattformen Devs nichts zahlen
- **>$200K** bereits an Vendors ausgezahlt (eigene Angabe)
- **Free-Tier: 25.000 Requests/Monat, keine Kreditkarte**
- One-click Install für Claude / Cursor / VS Code / Windsurf / Zed / ChatGPT
- **Unified Gateway:** Ein API-Key für viele Server. Eine Rechnung (Consolidated Billing) für alle Subscriptions
- **7-Dimensions-Audit:** Protocol Compliance, OAuth-Checks, Reliability-Benchmarks, Safety-Scanning — "Verified Badge"
- **Discovery API:** Andere Agents können den Marketplace programmatisch durchsuchen
- "Deploy in 30 seconds" + Scale-to-Zero + Instant Rollback
- Vergleichstabelle gegen "Other Directories" steht offen auf der Landing (Smithery/Glama nicht namentlich, aber gemeint)

**Pricing** (vom Landing-Hinweis):
- **Free:** 25K Requests/Mo, no credit card
- **Paid:** Nicht im Detail auf Landing — auf Pricing-Page checken
- **Creator-Side:** 80% Revenue-Share, Pay-per-Use oder Subscription pro Server

**Threat-Hypothese (vor dem Sprint):** **Niedrig bis Mittel.** MCPize konkurriert um Publisher (Creator-Economy-Story), nicht um A2A-Routing. Wenn a2a-router selbst Marketplace ist → direkter Konflikt. Wenn a2a-router Routing/Trust-Layer ist → komplementär. **80%-Story ist trotzdem ein guter Benchmark, denn a2a-router könnte sie unterbieten oder übertrumpfen.**

## Was kopieren — Was anders machen

**Übernehmen für a2a-router:**
- **Creator-Monetization als First-Class-Feature** (Smithery hat keine, Glama nur indirekt). Wenn a2a-router Agents listet, sollten Agent-Anbieter verdienen.
- **Sichtbare Quality-Audits** (7 Dimensions, Verified Badge) — Trust-Signal
- **Unified Gateway + Consolidated Billing** — extrem nutzerfreundlich
- "Free Tier ohne Credit Card" als Friction-Killer

**Bewusst anders / besser machen:**
- Kein "Build, Deploy, Earn"-Branding wenn Routing-First — eigene klare Story
- A2A statt MCP-Marketplace (es ist nicht das gleiche)
